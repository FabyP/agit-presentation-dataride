  • Those are the results
  • SAPUI5 got a score of 30, while React got a score of 95. the results depend on your testing environment, if you wanna recreate this, your results may differ
  • Quite the different, isnt it?
  • In SAPUI5, it takes 5 seconds till we're able to see first content and we have to wait further 4 seconds to be able to interact with the application. Makes in total 9 seconds till the user can start effectively working with our Fiori App. And in our other application, whick looks completely the same and provides the same functionality, youve seen it, the site is fully loaded after 2,6 seconds.
  • The reason for this, is, that in SAPUI5, we have a ton of unused CSS that is loaded, even for controls we havent even used. And there is the SAP ui core library, which somehow blocks rendering.
  • On the contrary, we have a react, a framework that comes with much less overhead and as a developer, i was able to specify, which resources i want to have loaded in my application
  • Since react is extremely lightweight, you need a lot of third party libraries to have a fully functioning application
  • Moreover, react uses a virtual dom, so if the state of one of my UI components changes, those actions are executed on a virtual dom. React then compares my changed virtual dom with my real dom and only changes those elements, which differ from the real dom

Other striking differences

  • Architecture
  • Need of CSS
  • In SAPUI5, we have a strict distribution into layout, functionality and model (MVC), while in react, we have all three of them combined in one component. This makes them reusable, but also worse to maintain the bigger my application gets, especially cause react lacks clearly defined best practices
  • And the other striking difference is the need off css in react, actually you need a ton of it, cause there arent much predifeined components or layouts you can use like in SAPUI5

Conclusion

  • SAPUI5 will stay your first choice for everything enterprise related. Due to the scope, the maintainability and better access to documentation
  • But if youre application can be clssified in the consumer environent, then the Ui5 web components with your framework of choice would be a better choice, cause they provide more flexibilty and have a better performance. Cause you all know, if my site takes to long to load, the consumer will just click the back button and chose a better performing site.
  • It really depends on the requirements of your project
